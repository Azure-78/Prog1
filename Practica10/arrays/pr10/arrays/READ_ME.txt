These files contain a battery of tests designed to verify the correct operation
of a program implementing the May version of project Televote

It assumes the existence of a Makefile with rules capable of generating a
a target with a program name = televote

Given that the program generated by the Makefile is named "televote", the
correct form of running the battery of tests is:


./protocolo.pl televote


The 'protocolo.pl" file is a Perl script.

If you don't have Perl installed in your system, you must install it:

sudo apt install perl

The mentioned script must have execute permissions. That is, if, after
extracting that file from the compressed tarball, it is not executable:

user@host (folder)>ls -l *.pl
-rw------- 1 user group 9758 30 abr 08:51 protocolo.pl

It is necessary to give it execute permissions:

user@host (folder)>chmod 700 *.pl
user@host (folder)>ls -l *.pl
-rwx------ 1 user group 9758 30 abr 08:51 protocolo.pl


The protocol runs the program repetitively, with this command:
  
./televote < protoXX.txt > my_outputXX.txt
  
that is, redirecting to its standard input, instead of the keyboard, the
contents of a file (protoXX.txt) containing input for the program, and
redirecting its output, instead of to the screen, to a file (my_outputXX.txt),
that will be afterwards evaluated.

Moreover, the contents of the succesive database files generated by the
program ("countries.txt" and "votes.txt") are also evaluated, if necessary.

After each test (with number XX), the resulting files are renamed as follows

- countries.txt -> my_countriesXX.txt
- votes.txt -> my_votesXX.txt

The correct output of the protocol must display the following:

of the Televote project

The program is called: "televote"

Cleaning up the environment
rm -f televote *.o *.bak ~*

Building the application
gcc -Wall -c database.c
gcc -Wall -c access.c
gcc -Wall access.o database_e.o televote.c -o televote
